### 1.3 Текущая архитектура данных (Encyclopedia)

Иерархия каталога в `EncyclopediaCatalog.kt`:

```
EncCountry (страна-производитель)
  └── EncBrand (марка: ВАЗ, BMW, Toyota...)
       └── EncPlatform (поколение/платформа: E90, PQ35, MQB...)
            └── EncSystem (система: ЭБУ, CAN, Иммо...)
                 └── subsystems: List<String> (подсистемы)
```

**Текущий охват каталога:**
- 6 стран (Россия, Германия, Япония, США/Корея, Китай, Франция/Италия)
- ~25 марок с детализацией
- ~50 платформ с топологией электроники
- ~200 системных разделов
- Подсистемы — текстовые строки (не структурированы)

### 1.4 Текущий UX Diagnosis Flow

**Wizard из 3 шагов:**
1. `StepCar` — ввод марки/модели/года + «Мой гараж» (недавние авто)
2. `StepSystem` — выбор одной из 13 систем через FlowRow
3. `StepSymptoms` — описание симптомов (текст/голос) + OBD2-коды + предопределённые теги + подсказки из локальной БД

**Результат (`StepResult`):**
- AI-генерация через `AiService` (Gemini → OpenAI → Pollinations fallback)
- JSON-парсинг ответа в `DiagnosisResponse` (solutions → steps)
- Чекбоксы для отслеживания выполнения шагов
- Голосование (like/dislike) для community-решений
- Предложение в глобальную базу (push to cloud)
- Сохранение в закладки

**Сильные стороны текущего UX:**
- Тёмная тема по умолчанию (DarkBackground `#0D0D0D`)
- Amber-акценты для контрастности
- Haptic feedback при действиях
- Голосовой ввод для грязных рук
- 56dp кнопки (достаточный tap target)
- Пошаговый wizard снижает когнитивную нагрузку
- AI-анимация загрузки с прогресс-баром

**Слабые стороны текущего UX:**
- Нет глобального поиска (поиск только внутри wizard)
- Нет тёмной/светлой темы — только dark (нет переключателя)
- OBD2-коды вводятся вручную (нет сканера/авто-определения)
- Нет offline-режима для контента энциклопедии (только cloud)
- Каталог энциклопедии — 6 кликов до статьи (Страна → Марка → Платформа → Система → Подсистема → Статья)
- Нет quick search по OBD2-кодам
- Нет визуальных распиновок

### 1.5 Текущий парсер (Parser Pipeline)

```
Drive2/AutoBK → Playwright (headless Chromium) → BeautifulSoup → AI (g4f/gpt-4o) → Firebase moderation_queue
```

**Компоненты парсера:**
- `spider_core.py` — базовый паук с asyncio + Playwright
- `drive2_spider.py` — парсер Drive2 (сбор ссылок по ключевым словам)
- `autobk_spider.py` — парсер AutoBK
- `ai_processor.py` — LLM-структурирование через g4f (gpt-4o)
- `firebase_uploader.py` — загрузка в Firestore `community_solutions`
- `database.py` — SQLite для отслеживания посещённых URL

**Проблемы парсера:**
- g4f不稳定 — зависит от сторонних прокси
- Нет валидации качества ответов LLM (кроме REJECT)
- Нет дедупликации контента
- Нет обратной связи от пользователей в парсер
- Лимит 3 concurrent tasks — медленно

```
│                                                  │
│  ┌─ 💡 Подсказка ──────────────────────────┐    │  ← раскрывающийся блок
│  │  Если реле щёлкает, но стартер не       │    │
│  │  крутит → проверьте обгонную муфту      │    │
│  │  и щётки стартера                       │    │
│  └──────────────────────────────────────────┘    │
│                                                  │
├──────────────────────────────────────────────────┤
│  📊 3/7  │  ◀ Назад  │  Вперёд ▶  │  🔄 Заново  │  ← sticky bottom
└──────────────────────────────────────────────────┘
```

**Ключевые UX-решения для дерева:**
1. **Один вопрос на экран** — нулевая когнитивная нагрузка
2. **Кнопки 60dp** — удобно нажимать грязными руками
3. **Прогресс-бар** — видно, сколько осталось
4. **Sticky bottom bar** — навигация всегда доступна
5. **Swipe** — влево = назад, вправо = вперёд
6. **Подсказка** — раскрывается по тапу, не перегружает
7. **Haptic** — вибрация при ответе
8. **Счётчик оставшегося времени** — мотивация пройти до конца

## ЧАСТЬ 3: СТРУКТУРА КАТАЛОГИЗАЦИИ

### 3.1 Оптимальная иерархия данных

```
Vehicle
├── Make (Марка)
│   └── Model (Модель)
│       └── Generation (Поколение/Год)
│           └── Engine (Двигатель) [опционально]
│               └── System (Система)
│                   └── Component (Узел/Подсистема)
│                       ├── Symptom (Симптом)
│                       ├── TroubleCode (DTC)
│                       ├── WiringDiagram (Распиновка)
│                       └── DiagnosticTree (Дерево диагностики)
```

**Текущая структура в `EncyclopediaCatalog.kt`:**
```
Country → Brand → Platform → System → Subsystem (text list)
```

**Проблемы текущей модели:**
1. `Country` — избыточный уровень (пользователь думает «мой бренд», а не «моя страна»)
2. `subsystems: List<String>` — не структурированы, нет привязки к DTC/симптомам
3. Нет привязки к конкретным двигателям (N47 vs N57 для BMW — разная электрика)
4. Нет `WiringDiagram` как отдельной сущности

**Рекомендация:** Перейти к модели с фасетным фильтром, убрав Country:

```
Make → Model → Generation → System → Component → Articles
                                        ├── DTC Codes
                                        ├── Symptoms
                                        ├── Wiring Diagrams
                                        └── Diagnostic Trees
```

### 3.2 Система тегирования

Каждая запись тегируется по **4 осям**:

| Ось | Теги | Примеры |
|---|---|---|
| **OBD2 DTC** | P/B/C/U-коды | `P0300`, `P0171`, `U0100`, `B1234` |
| **Система** | Подсистема авто | `starter`, `ecu.engine`, `immobilizer`, `can.bus` |
| **Симптом** | Нормализованные описания | `no-crank`, `crank-no-start`, `rough-idle`, `intermittent` |
| **Марка/Модель** | VIN-декод, поколение | `bmw.e90.2006-2011`, `toyota.corolla.e210` |

**Фасетный фильтр:** пользователь комбинирует оси. Пример: `BMW + E90 + immobilizer + P1602` → точный набор решений.

### 3.3 Интеллектуальный поиск (slang-aware)

**Словарь сленга → нормализация:**

```json
{
  "не крутит стартер": "starter.no-crank",
  "щёлкает но не крутит": "starter.relay-clicks-no-crank",
  "кидает на ходу": "engine.misfire.on-load",
  "чихает": "engine.misfire.at-idle",
  "пробивает массу": "ground.fault",
  "глючит иммо": "immobilizer.malfunction",
  "пропадает связь с блоком": "can.bus.no-communication",
  "лампа Check Engine": "indicator.check-engine-on",
  "масленка горит": "indicator.oil-pressure-warning",
  "башмачит стартер": "starter.slow-crank",
  "воет генератор": "generator.whine",
  "плавают обороты": "engine.rough-idle"
}
```

**Реализация:**
1. **Algolia/ElasticSearch** с кастомным синонимическим словарём (500+ пар)
2. **Phonetic matching** для опечаток
3. **Fuzzy search** с порогом 0.7
4. **Auto-suggest** после 2 символов
5. **LLM-реранжирование** топ-5 через lightweight LLM (Gemini Flash)
6. **Feedback loop:** кнопка «Это не то, что я искал» → корректировка весов

### 5.3 Конкретные категории для парсинга (первый-wave)

**Wave 1 (Сразу):**
1. **OBD2 DTC-каталог** — импорт готовой базы 5000+ кодов (P/B/C/U)
